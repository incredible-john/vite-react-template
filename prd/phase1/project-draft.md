# 项目概述
这个项目叫 Juolingo，是模仿 Duolingo 的一个英语学习软件。所以我希望你从各方面尽可能还原 Duolingo 原有的交互逻辑

# 技术栈
React, Tailwind CSS, shadcn/ui, Cloudflare Workers, Hono, Vite, Cloudflare D1 Database, Drizzle ORM

详细的情况，你可以查看整个项目结构

# 课程结构
课程由大到小有几个概念：

1. 主题 (Subject)
   (a) 包含：名人传记、童话故事、新闻、时事等各种主题
2. 单元 (Unit)
   (a) 比如：贝多芬、诺贝尔、南丁格尔
3. 课程 (Lesson)
   (a) 每个单元下分为不同的课程
4. 题目 (Challenges)
   (a) 课程下具体的题目

举个例子：
贝多芬这篇英语文章有 50 个新单词，那么有可能会分成 5 节课（5 个 Lesson），然后每个 Lesson 下面会学习 10 个新单词。
其中题目又有不同的类型，比如说：
1. 根据英文选中文意思
2. 左边一列有多个英文，右边一列有多个中文，然后互相匹配
3. 给出中文或英文句子，然后选取对应的翻译
反正有不同的题型，具体参考文件夹下的截图

# 页面路由
以下是可能存在的页面路由，我可能想得不够全面，你可以补充：

1. 主题（Subject）选择页面
   比如说可以选择名人传记或者童话故事。

2. 单元和课程展示页
   用户在这个页面可以看到不同的单元，每个单元下面有不同的课程，点击课程即可开始学习。

3. 学习（Learn）页面
   这就是具体的答题页面。用户会被展示不同类型的题目（Challenges）并进行回答，最终完成这一节课程。

4. 管理（Admin）页面
   用于添加主题、单元、课程、题目等内容。

请注意，Admin 页面需要在网页端（即宽屏设备）上操作，其他页面都是在移动端（即手机或者平板电脑）上操作。

# 要点

1. 编写 seed 脚本，用来填充一些初始数据，验证功能是否正常
2. 我们需要支持音频功能。比如说，页面上的题目是一个英文句子时，要能够点击并播放这句话的音频。此外，我们会对英文句子进行分词，分成不同的单词或词组。每个单词或词组下面会有虚线，当用户点击时，能够展示其释义并播放对应的音频。
3. 游戏页面 UI 尽可能还原 Duolingo 那种游戏化的设计语言，色彩丰富，动画流畅
4. 你要根据这种课程结构设计好数据库结构
